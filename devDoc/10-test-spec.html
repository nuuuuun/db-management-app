テスト戦略

テスト種別対象ツール実行タイミング
単体テストバックエンドクラス（JWT・権限チェック等）JUnit 5 + Spring Security Testコミット前・CI
結合テストAPI エンドポイント（実DBを使用）Spring Boot Test + TestRestTemplatePR マージ前
フロントエンドテストnpm audit（依存脆弱性）npm auditビルド時

セキュリティテスト権限昇格・テーブルアクセス・認証バイパス手動 + curlリリース前・改修後
⚠ 現状: Maven Wrapper 修正により mvnw でのテスト実行が可能になった。テストクラスは Spring Boot Test スターターが含まれているが、実際のテストケースは今後追加が必要。

シナリオテスト仕様

ST-01 基本業務フロー

手順操作確認ポイント
1admin でログインホーム画面が表示される。ユーザー管理・マスク設定メニューが見える。
2PROJECTSテーブルを開くデータが一覧表示される。ページング・検索・ソートが動作する。
3新規行を追加フォームで入力・保存後、一覧に反映される。
4追加した行を編集確認画面で変更前後が表示される。保存後に反映される。
5追加した行を削除確認後に削除され一覧から消える。
6CSVエクスポート → インポートエクスポートしたCSVを別テーブルにインポートできる。
7viewer でログイン追加・編集ボタンが非表示。エクスポートボタンが非表示。

ST-02 権限制御フロー

手順操作確認ポイント
1editor でログインユーザー管理・マスク設定メニューが非表示。
2直接 /users にアクセス403 またはリダイレクト。
3ADMIN がカラムをマスク設定editor でログインするとそのカラムが非表示になる。
4マスク設定を解除editor でカラムが再表示される。

セキュリティテスト仕様

SEC-01 テーブルアクセス制御

# allowlist外テーブルへのアクセス試行
curl -X GET http://localhost:18080/api/tables/USERS \
  -H "Authorization: Bearer $TOKEN"
# → 400 "対象外のテーブルです" を確認

curl -X GET http://localhost:18080/api/tables/MASKED_COLUMNS \
  -H "Authorization: Bearer $TOKEN"
# → 400 を確認

SEC-02 権限昇格テスト

# EDITOR トークンで ADMIN権限操作試行
curl -X POST http://localhost:18080/api/users \
  -H "Authorization: Bearer $EDITOR_TOKEN" \
  -H "Content-Type: application/json" \
  -d '{"username":"hacker","password":"test","role":"ADMIN"}'
# → 403 を確認

# EDITOR が自分のロールを ADMIN に変更試行
curl -X PUT http://localhost:18080/api/users/2 \
  -H "Authorization: Bearer $EDITOR_TOKEN" \
  -H "Content-Type: application/json" \
  -d '{"role":"ADMIN"}'
# → 403 を確認

SEC-03 CSVインポート制御

# EDITOR が overwrite モードでインポート試行
curl -X POST http://localhost:18080/api/import/PROJECTS?mode=overwrite \
  -H "Authorization: Bearer $EDITOR_TOKEN" \
  -F "file=@test.csv"
# → 403 を確認

SEC-04 トークンなしアクセス

# 認証ヘッダーなし
curl -X GET http://localhost:18080/api/tables
# → 401 を確認

# 無効なトークン
curl -X GET http://localhost:18080/api/tables \
  -H "Authorization: Bearer invalid.token.here"
# → 401 を確認
