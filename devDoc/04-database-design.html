テーブル概要

テーブル名日本語名区分説明
projects案件管理業務プロジェクト・案件の基本情報
requirements要求管理業務案件に紐づく要求一覧
specifications要件管理業務要求から展開した要件詳細
applicationsアプリ管理業務アプリケーションのバージョン・デプロイ状況
environments環境管理業務開発・ステージング・本番の環境情報
usersユーザー管理認証・認可ユーザー情報
masked_columnsマスク設定管理ロール別カラム非表示設定
operation_history操作履歴管理全データ変更の監査ログ
⚠ アクセス制御: 業務テーブル（上5件）のみ汎用APIからの操作を許可。 管理テーブルは専用APIまたはDirect DBアクセスでのみ操作可能。

[applications] [environments] app_id (PK) env_id (PK) app_name kankyo diff_version ... cvos_version kts_version ... [users] [masked_columns] [operation_history] id (PK) id (PK) id (PK) username (UNIQUE) table_name operated_at email column_name operator password (BCrypt) role target_table role target_id operation before_value after_value ※ FK制約なし。anken_name等は論理的な参照（画面リンクのみ）。

初期データ投入

アプリ起動時に DataInitializer.java が users テーブルの件数を確認し、0件の場合のみ以下の初期ユーザーを作成する。

ユーザー名ロール初期パスワード
adminADMIN環境変数 INIT_PASSWORD の値（デフォルト: password123）
editorEDITOR同上
viewerVIEWER同上
⚠ 本番運用前: 初期パスワードを必ず変更すること。INIT_PASSWORDは.envファイルで設定する。

DDL（schema.sql 抜粋）

-- 業務テーブル（例: projects）
CREATE TABLE IF NOT EXISTS projects (
    anken_id    SERIAL PRIMARY KEY,
    anken_name  VARCHAR(255),
    status      VARCHAR(50),
    jira_link   TEXT,
    tantou      VARCHAR(255),
    start_date  DATE,
    end_date    DATE,
    remarks     TEXT
);

-- システムテーブル
CREATE TABLE IF NOT EXISTS users (
    id       BIGSERIAL PRIMARY KEY,
    username VARCHAR(255) NOT NULL UNIQUE,
    email    VARCHAR(255),
    password VARCHAR(255) NOT NULL,
    role     VARCHAR(50)  NOT NULL
);

CREATE TABLE IF NOT EXISTS masked_columns (
    id          BIGSERIAL PRIMARY KEY,
    table_name  VARCHAR(100) NOT NULL,
    column_name VARCHAR(100) NOT NULL,
    role        VARCHAR(50)  NOT NULL
);

CREATE TABLE IF NOT EXISTS operation_history (
    id           BIGSERIAL PRIMARY KEY,
    operated_at  TIMESTAMP NOT NULL,
    operator     VARCHAR(255),
    target_table VARCHAR(100),
    target_id    VARCHAR(50),
    operation    VARCHAR(50),
    before_value TEXT,
    after_value  TEXT
);
