開発環境構築手順

リポジトリをクローン git clone [リポジトリURL]
cd db-management-app
.env ファイルを作成

.env.example をコピーして値を設定する。

copy .env.example .env # .env を編集して DB_USERNAME, DB_PASSWORD, JWT_SECRET を設定
PostgreSQL のみ Docker で起動（開発用） docker compose up db -d
バックエンドを起動 cd db-management
./mvnw spring-boot:run
フロントエンドを起動 cd frontend
npm install
npm run dev
# → http://localhost:5173 でアクセス

本番環境構築手順

サーバーに Docker + Docker Compose をインストール # Ubuntu/Debian の例 sudo apt-get update sudo apt-get install docker.io docker-compose-v2 sudo systemctl enable --now docker sudo usermod -aG docker $USER
リポジトリをクローン git clone [リポジトリURL]
cd db-management-app
.env ファイルを作成（本番用） cp .env.example .env # 本番用の強固なパスワード・秘密鍵を設定する # DB_USERNAME=（任意のユーザー名） # DB_PASSWORD=（強固なパスワード） # JWT_SECRET=（32文字以上のランダム文字列） # INIT_PASSWORD=（初期ユーザーパスワード）
ビルド・起動 docker compose up -d --build
動作確認 # コンテナ状態確認 docker compose ps # アプリログ確認 docker compose logs app # ブラウザで http://[サーバーIP]:18080 にアクセス # admin / [INIT_PASSWORDの値] でログイン
初期パスワード変更

admin ユーザーでログイン後、ユーザー管理画面から全ユーザーのパスワードを変更する。

日常運用手順

アプリの起動・停止

# 起動
docker compose up -d

# 停止（データは保持）
docker compose down

# 再起動
docker compose restart app

# ログ確認
docker compose logs -f app

アップデート手順

# 1. コードを取得
git pull origin main

# 2. ビルド・再起動（ダウンタイム最小化）
docker compose up -d --build

# 3. 動作確認
docker compose ps
docker compose logs app

コンテナ・DB状態確認

# コンテナ一覧
docker compose ps

# DBに直接接続
docker compose exec db psql -U $DB_USERNAME -d dbmanagement

# テーブル一覧確認
\dt

# 接続ユーザー確認
SELECT * FROM users;

バックアップ・リストア

バックアップ（推奨: 毎日実行）

# データベースのダンプ
docker compose exec db pg_dump -U $DB_USERNAME dbmanagement \
  > backup_$(date +%Y%m%d).sql

# 圧縮保存
gzip backup_$(date +%Y%m%d).sql

バックアップ自動化（cron）

# crontab に追記（毎日 2:00 AM）
0 2 * * * cd /path/to/db-management-app && \
  docker compose exec -T db pg_dump -U $DB_USERNAME dbmanagement \
  | gzip > /backup/db_$(date +\%Y\%m\%d).sql.gz

リストア手順

# 1. バックアップファイルを解凍
gunzip backup_YYYYMMDD.sql.gz

# 2. DBを初期化してリストア
docker compose exec db psql -U $DB_USERNAME -d dbmanagement \
  < backup_YYYYMMDD.sql

# または新規DBへリストア
docker compose exec db psql -U $DB_USERNAME -c "DROP DATABASE dbmanagement;"
docker compose exec db psql -U $DB_USERNAME -c "CREATE DATABASE dbmanagement;"
docker compose exec db psql -U $DB_USERNAME -d dbmanagement < backup_YYYYMMDD.sql

障害対応手順

アプリにアクセスできない

確認項目コマンド対処
コンテナ起動確認docker compose psExited なら docker compose up -d で再起動
ポート確認netstat -tlnp | grep 18080ポート競合なら docker-compose.yml のポートを変更
アプリログ確認docker compose logs app --tail=100エラー内容に応じて対処
DB接続確認docker compose exec db pg_isreadyDB再起動: docker compose restart db

DB接続エラー

# DB コンテナの状態確認
docker compose ps db
docker compose logs db --tail=50

# DB コンテナ再起動
docker compose restart db

# アプリも再起動
docker compose restart app

データが消えた（誤操作・バグ）

操作履歴で確認 docker compose exec db psql -U $DB_USERNAME -d dbmanagement \ -c "SELECT * FROM operation_history ORDER BY operated_at DESC LIMIT 50;"
直近バックアップからリストア

上記の「リストア手順」を参照。直近のバックアップファイルを使用する。

ディスク容量不足

# Dockerのディスク使用量確認
docker system df

# 未使用イメージ・コンテナを削除
docker system prune -f

# バックアップファイルの古いものを削除
ls -lh /backup/
rm /backup/db_YYYYMMDD.sql.gz

障害対応チェックリスト

  • コンテナが起動しているか確認（docker compose ps）
  • アプリのログにエラーが出ていないか確認（docker compose logs app）
  • DBのログにエラーが出ていないか確認（docker compose logs db）
  • ディスク容量が不足していないか確認（df -h）
  • ネットワーク接続（LAN）に問題がないか確認
  • ポート18080 が他のプロセスに使われていないか確認
  • .env ファイルが存在し、必要な変数が設定されているか確認
  • 障害内容・対処内容を記録する
