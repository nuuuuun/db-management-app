セキュリティ実装詳細

JwtFilter の処理順序

doFilterInternal():
  1. Authorization ヘッダー取得
  2. "Bearer " プレフィックス確認
  3. JwtUtil.extractUsername() でユーザー名取得
  4. JwtUtil.validateToken() でトークン検証
  5. UserDetails を生成し SecurityContextHolder にセット
  6. ロール情報を SimpleGrantedAuthority として付与
  7. chain.doFilter() で次のフィルタへ

SecurityConfig の主要設定

http
  .csrf(csrf -> csrf.disable())
  .sessionManagement(session -> session
      .sessionCreationPolicy(SessionCreationPolicy.STATELESS))
  .authorizeHttpRequests(auth -> auth
      .requestMatchers("/api/auth/login", "/", "/index.html",
                       "/assets/**", "/favicon.ico").permitAll()
      .anyRequest().authenticated())
  .addFilterBefore(jwtFilter, UsernamePasswordAuthenticationFilter.class)
  .cors(cors -> cors.configurationSource(corsConfigurationSource()))

データアクセス設計

JPA vs JdbcTemplate の使い分け

技術使用箇所理由
Spring Data JPAUser / MaskedColumn エンティティスキーマ固定・型安全なCRUDが可能
JdbcTemplate業務テーブル操作・操作履歴テーブル名・カラム名が実行時に決まるため動的SQL必須
DatabaseMetaDataテーブル一覧・カラム一覧取得スキーマ情報をコードに持たず動的に取得
