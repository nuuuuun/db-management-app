ナビゲーションガード

// router/index.js
router.beforeEach((to, from, next) => {
  if (to.path !== '/login' && !isLoggedIn()) {
    next('/login');
  } else {
    next();
  }
});

API連携

Axiosインスタンス（api/index.js）

const api = axios.create({
  baseURL: '/api'  // Vite proxy or Spring Boot 同一オリジン
});

// リクエストインターセプター: JWTを自動付与
api.interceptors.request.use(config => {
  const token = getToken();
  if (token) config.headers.Authorization = `Bearer ${token}`;
  return config;
});

// レスポンスインターセプター: 401 でログアウト
api.interceptors.response.use(
  res => res,
  err => {
    if (err.response?.status === 401) {
      logout();
      router.push('/login');
    }
    return Promise.reject(err);
  }
);

開発時のProxy설정（vite.config.js）

// 開発時: localhost:5173 → localhost:8080 へプロキシ
server: {
  proxy: {
    '/api': 'http://localhost:8080'
  }
}

主要ビューの詳細

TableDetailView — メイン操作画面

機能実装
ページングpage・size をクエリパラメータでAPI送信。10/20/50/100件切替。
検索キーワード入力 → search パラメータ送信。全カラム対象。
ソートヘッダークリック → sortCol・sortDir パラメータ送信。2回クリックで反転。
行追加インラインフォーム表示 → POST /api/tables/{name}/rows
行編集/tables/{name}/edit/{id} へ遷移
行削除確認ダイアログ → DELETE /api/tables/{name}/rows/{id}
ExcelエクスポートGET /api/export/excel/{tableName} でファイルダウンロード
マスク適用GET /api/masked-columns でマスク設定取得→カラム非表示
クロステーブルリンクCOLUMN_LINKS（utils/labels.js）定義に従い、値をリンクとして表示

TableRowEditView・TableRowEditConfirmView

フロー: 編集フォーム（TableRowEditView）→ 確認画面（TableRowEditConfirmView）→ PUT /api/tables/{name}/rows/{id}

確認画面では変更前・変更後の値を対比表示する。確定ボタンで実際のAPI呼び出しを行う。

CsvBulkView — 一括入出力

機能実装
エクスポートGET /api/export/all → blob でファイルダウンロード
インポートファイル選択 → POST /api/import/all （multipart）
結果表示テーブルごとの inserted 件数・errors を一覧表示

ユーティリティ詳細

auth.js — 認証状態管理

関数説明
setToken(token)localStorage に JWT を保存
getToken()localStorage から JWT を取得
removeToken()localStorage から JWT を削除
isLoggedIn()トークンが存在するか確認
setUser(user)ユーザー情報（username, role）を localStorage に保存
getUser()ユーザー情報を取得
logout()token・user を localStorage から削除

labels.js — UI定義

定数内容
TABLE_LABELSテーブル名→日本語名マッピング（例: projects → "案件管理"）
COLUMN_LABELSカラム名→日本語名マッピング（例: anken_name → "案件名"）
FIELD_DEFSカラム名→入力タイプ定義（date / select / text）
COLUMN_LINKSクロステーブルリンク定義（例: REQUIREMENTS.ANKEN_NAME → /tables/projects）

ビルド・デプロイ

本番ビルド手順

# フロントエンドビルド
cd frontend
npm ci
npm run build
# → dist/ に静的ファイルが生成される

# dist/ を Spring Boot の静的リソースにコピー
cp -r dist/* ../db-management/src/main/resources/static/

# Dockerfile のマルチステージビルドでは自動化されている

開発サーバー起動

# フロントエンド開発サーバー（HMR有効）
cd frontend
npm run dev
# → http://localhost:5173 で起動
# → /api/* は http://localhost:8080 へプロキシ

# バックエンド開発起動（別ターミナル）
cd db-management
./mvnw spring-boot:run

本番環境では

docker compose up -d --build を実行するだけでOK。Dockerfileのマルチステージビルドが:

  1. フロントエンドをビルド（npm run build）
  2. dist/ を Spring Boot の static/ にコピー
  3. Spring Boot をビルド（mvnw package）
  4. 実行イメージを作成

ユーザーは http://[サーバーIP]:18080 にアクセスするだけで利用可能。
