CSV単体インポート仕様

処理フロー

ファイル受信
→
権限チェック
→
テーブル名検証（allowlist）
DBカラム取得
→
CSVヘッダー検証
→
全件読み込み
overwriteなら DELETE実行
→
INSERT（@Transactional）
→
結果返却

パラメータ

パラメータ型説明
tableName (path)string対象テーブル名。allowlist（5テーブル）以外はエラー。
file (form)MultipartFileUTF-8 CSV ファイル
mode (form)stringappend（デフォルト）: 既存データに追加 / overwrite: 全件削除後インポート（ADMIN のみ）

バリデーション

チェックエラー内容
テーブルがallowlist外400: "対象外のテーブルです"
CSVに存在しないカラム名400: "カラム '{カラム名}' は存在しません"
overwrite で非ADMIN403: "上書きモードは管理者のみ使用できます"
行単位のINSERTエラーエラー行を errors リストに記録して処理続行

レスポンス

{ "inserted": 15, "errors": ["行3: インポートエラーが発生しました"] }

CSV一括エクスポート/インポート仕様

フォーマット（独自拡張CSV）

複数テーブルを1ファイルにまとめるため、 #TABLE:[テーブル名] マーカーでセクション分割する。

#TABLE:PROJECTS
anken_id,anken_name,status,jira_link,...
1,サンプル案件A,進行中,https://jira.example.com/XXX,...
2,サンプル案件B,完了,https://jira.example.com/YYY,...

#TABLE:REQUIREMENTS
yoqyu_id,anken_name,yoqyu_name,status,...
1,サンプル案件A,要求001,承認済,...

エクスポート処理

項目仕様
ファイル名db-export.csv
文字コードUTF-8
マスク適用ロール別に masked_columns を参照しカラムを除外
出力対象ALLOWED_TABLES に含まれる全業務テーブル

一括インポート処理

項目仕様
BOM除去ファイル先頭の BOM（﻿）を自動除去
セクション分割#TABLE: マーカーで文字列分割
空行スキップsetIgnoreEmptyLines(true) で空行を無視
エラー時テーブルごとの処理。1テーブルエラーでも他テーブルは継続。

Excelエクスポート仕様

出力仕様

項目仕様
ファイル形式.xlsx（OOXML形式）
ライブラリApache POI 5.2.5
シート構成（単体）1シート：テーブル名
シート構成（全体）業務テーブル分のシートをシート名で分割
ヘッダー行COLUMN_LABELS（utils/labels.js）の日本語名を使用
マスク適用ロール別マスク設定に従いカラムを除外
スタイルヘッダー行に背景色・太字スタイルを適用

共通バリデーションルール

対象チェック内容エラー
テーブル名ALLOWED_TABLES（PROJECTS/REQUIREMENTS/SPECIFICATIONS/APPLICATIONS/ENVIRONMENTS）に含まれること400
カラム名DBのメタデータから取得したカラムに含まれること400
モード指定overwrite は ADMIN ロールのみ許可403
ファイル形式UTF-8エンコード（BOM有りも対応）—
⚠ 注意: CSVインポートはDBの型チェックに依存。型不一致・制約違反は行単位のエラーとして記録され、その行のみスキップされる。詳細エラー情報はサーバーログに記録。

CSVフォーマット仕様

項目仕様
区切り文字カンマ（,）
文字コードUTF-8（BOM有り・無し両対応）
ヘッダー行1行目にカラム名（大文字・小文字どちらも可、トリム処理あり）
引用符ダブルクォート（"）。カンマ・改行・ダブルクォートを含む値に使用。
エスケープダブルクォートを2重（""）にエスケープ
空値空文字（""）または空フィールド → NULL として挿入
