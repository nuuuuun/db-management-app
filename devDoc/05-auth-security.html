認証フロー

1
ログインリクエスト
クライアントが POST /api/auth/login に { username, password } を送信
2
ユーザー検索・パスワード検証
AuthController → UserRepository.findByUsername() でDBからユーザー取得。BCryptPasswordEncoder.matches() でパスワード照合。
3
JWT生成・返却
認証成功時、JwtUtil.generateToken(username, role) でJWTを生成。{ "token": "eyJ..." } を返す。
4
トークン保管（クライアント側）
フロントエンドは localStorage にトークンを保存。全APIリクエストの Authorization: Bearer ヘッダーに付与。
5
リクエスト検証（JwtFilter）
JwtFilter が Authorization ヘッダーを取り出し → JwtUtil.validateToken() で署名・有効期限を検証 → SecurityContextHolder に認証情報をセット。
6
API処理
各コントローラが SecurityUtils.getCurrentRole() でロールを取得し、権限チェックを実施。

JWT仕様

トークン構成

項目値
署名アルゴリズムHS256（HMAC-SHA256）
有効期限86400000ms（24時間）。環境変数 JWT_EXPIRATION で変更可。
クレーム: subユーザー名
クレーム: roleADMIN / EDITOR / VIEWER
クレーム: iat発行日時
クレーム: exp有効期限
秘密鍵環境変数 JWT_SECRET。32文字以上のランダム文字列推奨。Gitにコミット禁止。

JWT関連クラス

クラス役割
JwtUtil.javagenerateToken() / validateToken() / extractUsername() / extractRole()
JwtFilter.javaOncePerRequestFilter。リクエストごとにトークン検証・SecurityContext設定。
SecurityUtils.javagetCurrentRole() / getCurrentUsername() をコントローラから呼び出す静的ユーティリティ。

認可・RBAC設計

ロール定義

ロール権限レベル主な制限
ADMIN最上位全機能利用可能
EDITOR中間ユーザー管理・マスク設定・上書きインポート不可
VIEWER最低データ参照のみ。書き込み・エクスポート全て不可

認可チェックの実装パターン

Spring Securityのアノテーションは使用せず、各コントローラで SecurityUtils.getCurrentRole() を直接チェック。

// ADMIN のみ許可
if (!"ADMIN".equals(SecurityUtils.getCurrentRole())) {
    return ResponseEntity.status(403).body(Map.of("error", "権限がありません"));
}

// VIEWER は書き込み禁止
if ("VIEWER".equals(SecurityUtils.getCurrentRole())) {
    return ResponseEntity.status(403).body(Map.of("error", "権限がありません"));
}

SecurityConfig のエンドポイント設定

パス設定
/api/auth/loginpermitAll（認証不要）
/（静的リソース）permitAll
/api/**（その他）authenticated（JWTトークン必須）

カラムマスク設計

仕組み

masked_columns テーブルに「テーブル名・カラム名・ロール」の組み合わせを登録することで、該当ロールのユーザーにはそのカラムのデータを返さない。

適用範囲: テーブルデータ取得API・CSV一括エクスポート・Excelエクスポート

ADMIN は常に全カラム表示。 マスク設定はADMINには適用されない。

マスク適用ロジック

// EDITOR の場合: EDITOR設定 のみ確認
// VIEWER の場合: VIEWER + EDITOR 設定を両方確認（累積マスク）
List<String> rolesToCheck = "VIEWER".equals(role)
    ? List.of("VIEWER", "EDITOR")
    : List.of("EDITOR");

Set<String> maskedCols = maskedColumnRepository
    .findByTableNameAndRoleIn(tableName, rolesToCheck)
    .stream().map(MaskedColumn::getColumnName)
    .collect(Collectors.toSet());

// データ取得後、maskedCols のカラムを除外して返す
