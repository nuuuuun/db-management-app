構成のポイント:

  • フロントエンドはビルド成果物（dist/）をSpring Bootの静的リソースとして組み込み。別途Webサーバー不要。
  • バックエンドとDBはDocker Composeで同一ホストで管理。
  • ユーザーは http://[サーバーIP]:18080 へアクセスするだけで利用可能。
  • 社内LAN限定運用のため、インターネット向けロードバランサ・HTTPS証明書は不要。

docker-compose.yml 要点

services:
  db:
    image: postgres:16-alpine
    environment:
      POSTGRES_DB: dbmanagement
      POSTGRES_USER: ${DB_USERNAME}
      POSTGRES_PASSWORD: ${DB_PASSWORD}
    ports: ["5432:5432"]
    volumes:
      - postgres_data:/var/lib/postgresql/data
    healthcheck:
      test: ["CMD-SHELL", "pg_isready -U ${DB_USERNAME}"]
      interval: 10s / timeout: 5s / retries: 5

  app:
    build: .
    ports: ["18080:8080"]
    env_file: .env
    environment:
      DB_URL: jdbc:postgresql://db:5432/dbmanagement
    depends_on:
      db: { condition: service_healthy }

volumes:
  postgres_data:

Dockerfile 要点（マルチステージビルド）

# Stage 1: フロントエンドビルド
FROM node:20-alpine AS frontend-build
WORKDIR /app/frontend
COPY frontend/ .
RUN npm ci && npm run build

# Stage 2: バックエンドビルド
FROM eclipse-temurin:21-jdk-alpine AS backend-build
WORKDIR /app
COPY db-management/ .
COPY --from=frontend-build /app/frontend/dist ./src/main/resources/static
RUN ./mvnw -B package -DskipTests

# Stage 3: 実行イメージ
FROM eclipse-temurin:21-jre-alpine
WORKDIR /app
COPY --from=backend-build /app/target/*.jar app.jar
ENTRYPOINT ["java","-jar","app.jar","--spring.profiles.active=prod"]
