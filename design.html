DB管理アプリ 設計書

作成日: 2026-05-04　　バージョン: 1.0　　対象環境: ローカル / 社内LAN

システム概要

業務用DBの内容をブラウザ上で閲覧・編集できるWebアプリ。社内・社外のユーザーが権限レベルに応じてCRUD操作・CSVインポート・Excelエクスポートを行える。

対象ユーザー

ロール人数種別
Admin少数社内管理者
Editor数人社内担当者
Viewer残り社外・閲覧専用

主な機能

機能説明
テーブル閲覧全テーブルの一覧・検索・ソート
CRUD行の追加・編集・削除
CSVインポートテーブル単位の一括登録
Excelエクスポートテーブル・全件のxlsx出力
カラムマスキング権限別にカラムを非表示
操作履歴全変更操作をDBに記録
ユーザー管理Adminによるアカウント管理

認証・権限

認証方式

JWT（JSON Web Token）によるステートレス認証。ログイン時にトークンを発行し、以降のリクエストは Authorization: Bearer {token} ヘッダーで認証する。パスワードはBCryptでハッシュ化。

カラムマスキング

テーブル × カラム × ロールの組み合わせで非表示を設定。APIレスポンス生成時にJSONから除外して返す（*置換ではなくカラム自体を削除）。

-- masked_columns テーブル
table_name  : "PROJECTS"
column_name : "SALARY"
role        : "VIEWER"   -- or "EDITOR"

起動方法

前提条件

ソフトウェアバージョン確認方法
Docker Desktop最新docker --version
Git任意git --version

初回起動

プロジェクトをサーバーPCに配置する（git clone またはフォルダコピー）

プロジェクトルートでコマンドを実行する

cd D:\path\to\db-management-app
docker compose up -d --build
# 初回は 5〜10分かかる（Dockerイメージのビルド）

ブラウザでアクセスする

# 起動したPCから
http://localhost:18080

# 他のPCから（社内LAN）
http://[サーバーPCのIPアドレス]:18080

サーバーPCのIPアドレスを確認し、他のユーザーに共有する（ipconfig で確認）

日常操作

操作コマンド
起動docker compose up -d
停止docker compose down
ログ確認docker compose logs -f app
DB確認docker compose exec db psql -U postgres dbmanagement
再ビルド（コード変更後）docker compose up -d --build
データはDockerのボリューム（postgres_data）に永続化されている。docker compose down してもDBのデータは消えない。
PC再起動後はDockerが自動起動する設定（Docker Desktop の "Start Docker Desktop when you sign in"）にしておくと便利。その場合 docker compose up -d も自動化したいなら、タスクスケジューラに登録する。

IPアドレス固定（推奨）

サーバーPCのLAN IPが変わるとURLが変わってしまう。ルーターのDHCP設定でサーバーPCのMACアドレスに固定IPを割り当てておくと安定する。

環境変数

docker-compose.yml の environment セクションで設定する。

変数名デフォルト値説明
DB_URLjdbc:postgresql://db:5432/dbmanagementPostgreSQL接続URL
DB_USERNAMEpostgresDBユーザー名
DB_PASSWORDpostgresDBパスワード（本番は変更必須）
JWT_SECRETdb-management-jwt-secret-key-for-dev-only-2026JWTトークン署名キー（本番は変更必須）
INIT_PASSWORDpassword123初期ユーザーのパスワード
本番・社内LAN運用時は DB_PASSWORD と JWT_SECRET を必ず変更すること。
