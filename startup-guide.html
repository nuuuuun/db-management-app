エンジニア以外の方向け　｜　わからないことがあれば担当者に連絡してください

🟢 起動方法を選んでください
✅ 現在はこちらを使ってください。 Docker不要・インストールされたPostgreSQLで動かします。
⏱️ 所要時間：初回は約5分（前提ソフトのインストール含む）、2回目以降は約1分です。
1前提ソフトが入っているか確認する（初回のみ）

以下の3つがインストール済みであることを確認してください。

ソフトウェア確認方法入っていない場合
PostgreSQL 16 スタートメニューに「pgAdmin」または「PostgreSQL」がある 担当者に依頼してインストールしてもらう
Node.js 20 コマンドプロンプトで node -v を実行 → バージョンが出る 担当者に依頼
JDK 21 コマンドプロンプトで java -version を実行 → バージョンが出る 担当者に依頼
2PostgreSQL が起動しているか確認する

スタートメニューで「サービス」と検索して開き、一覧に 「postgresql-x64-16」 があることを確認してください。

✅ 状態が「実行中」→ そのまま次へ
⚠️ 状態が「停止」→ 右クリック →「開始」をクリックする
3PowerShell を開く

Windowsキー を押して「PowerShell」と入力 → 「Windows PowerShell」 を右クリック →「管理者として実行」を選んでください。

💡 青いウィンドウが開けばOKです。
4アプリのフォルダに移動する

以下をコピーして PowerShell に貼り付け、Enter を押してください：

cd D:\プログラミング\git\db-management-app
💡 PowerShellへの貼り付けは 右クリック または Ctrl+V
5起動スクリプトを実行する

以下をコピー＆ペーストして Enter を押してください：

.\scripts\start-local.ps1

文字がたくさん流れます。「Started DbManagementApplication」 のような文字が出てきたら起動完了です。

⚠️ 初回は 2〜3分かかります（フロントエンドのビルド処理があります）。エラーが出なければ待っていてください。
6ブラウザでアクセスする

ブラウザのアドレスバーに以下を入力して Enter を押してください：

http://localhost:8080
http://localhost:8080
🔐 ログイン画面が表示されればOKです
✅ ログイン画面が出た → 起動成功！
❌ 「このサイトにアクセスできません」と出た場合 → 1〜2分待ってからもう一度試してください（アプリの起動中の場合があります）
📡 他のPCからアクセスする場合（社内LANのみ）

このPCのIPアドレスを確認して、他のPCのブラウザから http://[IPアドレス]:8080 でアクセスできます。

IPアドレスの確認方法：PowerShellで ipconfig と入力 → 「IPv4 アドレス」の数字（例: 192.168.1.10）

⚠️ Windowsファイアウォールで8080番ポートを開放する必要がある場合があります。担当者に相談してください。
⚠️ Docker起動は AWS移行時 に使用します。通常はローカル起動をご利用ください。
⏱️ 所要時間：初回は約10分、2回目以降は約1〜2分です。
1Docker Desktop が起動しているか確認する

画面右下のタスクトレイ（時計の近く）に クジラのアイコン 🐳 があれば起動済みです。

✅ クジラアイコンがある → そのまま次へ進む
⚠️ アイコンがない → スタートメニューから「Docker Desktop」を起動して、クジラアイコンが出るまで待つ
2.env ファイルを確認する（初回のみ）

アプリフォルダに .env ファイルが存在することを確認してください。

⚠️ ない場合は担当者に作成してもらってください。
3PowerShell を開いてフォルダに移動する
cd D:\プログラミング\git\db-management-app
4起動コマンドを実行する
docker compose up -d

「✔ Container ... Started」 が出てきたら完了です。

⚠️ 初回は 5〜10分かかります。
5ブラウザでアクセスする
http://localhost:18080
✅ ログイン画面が出た → 起動成功！
🔴 アプリを停止する手順
🗄️ 停止してもデータベースのデータは消えません。安心して停止できます。

💻 ローカル起動を停止する（通常）

1PowerShell ウィンドウで Ctrl+C を押す

起動スクリプトを実行した PowerShell ウィンドウを選択して Ctrl+C を押してください。

「Terminate batch job (Y/N)?」と聞かれたら Y を押してください。

✅ これだけで停止完了です。PostgreSQLはそのまま動き続けて問題ありません。

💻 アプリの使い方

ログイン

担当者から受け取ったユーザー名とパスワードでログインしてください。

ロールできること
管理者 すべての操作（追加・編集・削除・ユーザー管理）
編集者 閲覧・追加・編集・CSVインポート（削除は一部制限）
閲覧者 閲覧のみ

基本操作

  • 📋
    テーブルを見る：左のメニューからテーブル名をクリック → データが一覧表示されます
  • 🔍
    検索する：一覧画面の上部にある検索ボックスにキーワードを入力 → 自動で絞り込まれます
  • ➕
    データを追加する：「新規追加」ボタンをクリック → 入力フォームに記入 → 「保存」（編集者以上）
  • ✏️
    データを編集する：行をクリック → 「編集」ボタン → 修正して「保存」（編集者以上）
  • 🗑️
    データを削除する：行をクリック → 「削除」ボタン（管理者のみ）
  • 📥
    CSVを一括登録する：「CSVインポート」ボタン → ファイルを選択 → インポート実行（編集者以上）
  • 📊
    Excelに出力する：「Excelエクスポート」ボタンをクリック → .xlsxファイルがダウンロードされます

CSVインポートのルール

  • 📌
    CSVの1行目はカラム名（見出し行）である必要があります
  • 📌
    文字コードは UTF-8 で保存してください（Excelで保存する場合は「CSV UTF-8（コンマ区切り）」を選択）
  • 📌
    「上書き」モードを選ぶと既存データが全件削除されてからインポートされます。管理者のみ使用可能です。
❓ 困ったときのQ&A

💻 ローカル起動のトラブル

「スクリプトの実行が無効になっています」と出た ▼
PowerShell のスクリプト実行が制限されています。管理者として実行した PowerShell で以下を実行してから、もう一度スクリプトを実行してください：

Set-ExecutionPolicy RemoteSigned -Scope CurrentUser

「変更しますか？」と聞かれたら Y を押してください。
「PostgreSQL に接続できませんでした」と出た ▼
PostgreSQL サービスが起動していない可能性があります。

1. スタートメニューで「サービス」と検索して開く
2. 「postgresql-x64-16」を探す
3. 右クリック →「開始」を選ぶ

それでも解決しない場合は担当者に連絡してください。
ブラウザに「このサイトにアクセスできません」と表示される ▼
原因1：アプリがまだ起動中
スクリプト実行後、「Started DbManagementApplication」の表示が出るまで待ってから再度アクセスしてください。

原因2：起動スクリプトでエラーが出た
PowerShell の画面に赤いエラーメッセージが出ていないか確認してください。
エラーのスクリーンショットを担当者に送ってください。
PC を再起動したらアクセスできなくなった ▼
PC再起動後はアプリが停止します。「起動する」タブのローカル起動手順に沿ってもう一度起動してください。

※ PostgreSQL は自動起動するよう設定されていれば手動起動不要です。
ログインできない ▼
ユーザー名・パスワードを確認してください（大文字・小文字の区別があります）。
パスワードがわからない場合は管理者に再発行を依頼してください。
他のPCからアクセスできない ▼
以下を確認してください：

1. アプリが起動しているPCのIPアドレスを確認する（ipconfig コマンド → IPv4アドレス）
2. ブラウザで http://[サーバーのIPアドレス]:8080 にアクセスする
3. 同じ社内LANに接続しているか確認する
4. Windowsファイアウォールで8080番ポートをブロックしている場合があります。担当者にご連絡ください。
CSVインポートでエラーが出る ▼
よくある原因：
・1行目のカラム名がDBのカラム名と一致していない
・文字コードがUTF-8でない（Excelで「名前を付けて保存」→「CSV UTF-8（コンマ区切り）」を選択してください）
・必須項目が空になっている行がある

エラーが出た行番号がメッセージに表示されるので、該当行を確認してください。
📞 解決しない場合

エラーメッセージのスクリーンショットを撮って、システム担当者に連絡してください。
